*   **Architecture & Style:** Favor DRY, KISS (simple, single-purpose pure functions), and YAGNI. Check for existing logic before writing new code. No mutating inputs or global state. Use OOP classes only for external connectors/interfaces. English comments only. All imports at the top. Verify compilation and typing (e.g., running type-checks/linting) before reporting any task complete; do not assume successful file writes mean valid code.
